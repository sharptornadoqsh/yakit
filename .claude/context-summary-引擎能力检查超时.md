## 项目上下文摘要（引擎能力检查超时）

生成时间：2026-07-16 23:58:00

### 一、需求与验收条件

- 目标：解决双击 `start-enterprise-no-license-dev.cmd` 后界面停留在本地引擎检查阶段的问题。
- 范围：仅调整本地仓库引擎的能力检查等待时限，不处理前端编译警告，不改变引擎来源与更新策略。
- 验收：能力检查不会因十一秒的外层时限误杀；现有启动时限保持不变；相关契约测试通过；双击脚本能够使本地引擎进入启动流程。

### 二、现有实现分析

- `app/main/handlers/newEngineStatus.js` 的 `asyncAllowSecretLocal` 调用仓库引擎执行 `check-secret-local-grpc`，外层仅等待十一秒。
- 同一文件的本地引擎启动流程使用 `LOCAL_ENGINE_START_TIMEOUT_MS`，当前值为一百八十秒，覆盖首次数据库初始化耗时。
- 同一文件的数据库修复流程具有独立的十一秒时限，与当前能力检查不是同一任务，本次不改变。
- `app/renderer/engine-link-startup/src/pages/StartupPage/components/LocalEngine/index.tsx` 根据能力检查返回状态展示检查结果，主进程超时会阻止后续引擎启动。
- `app/main/__test__/product.test.js` 已采用源码契约测试保护本地引擎启动时限，可沿用该模式保护能力检查时限。

### 三、运行证据

- 最新引擎日志停在“检查本地随机密码模式支持”，随后记录“检查随机密码模式超时”，没有进入“开始启动引擎”。
- 同次界面日志证明两个开发服务器分别加载 `http://127.0.0.1:3000/` 与 `http://127.0.0.1:5173/`，因此前端编译警告不是停滞原因。
- 使用相同引擎、端口和用户目录手工执行 `check-secret-local-grpc` 成功，退出码为零，耗时约三点七秒，证明仓库引擎文件有效。
- 历史成功日志显示同一能力检查能够完成，而首次引擎数据库初始化约耗时五十六秒，说明问题具有冷启动时序特征。
- 动态复现时，五千一百七十三端口可立即返回，三千端口虽已监听但八秒内没有 HTTP 响应，Electron 因就绪检查未通过而尚未启动。
- 主渲染进程在“检查问题进行中”阶段占用约六吉字节内存，并存在两个类型检查报告进程，证明启动期类型检查是前端开发服务无响应的直接条件。

### 四、项目约定与复用

- 主进程使用 CommonJS、两空格缩进、单引号且不使用分号。
- 时限以文件顶部常量表达，并在日志中显示秒数，沿用 `LOCAL_ENGINE_START_TIMEOUT_MS` 的现有模式。
- 测试沿用 Vitest 源码契约断言，不新增依赖，不改变界面组件。
- `config-overrides.js` 已使用开发模式条件配置和插件列表改写；专用脚本可通过环境变量仅关闭当前开发会话的类型检查插件。

### 五、依赖与集成点

- 输入：界面传入的端口与产品版本。
- 进程：Electron 主进程通过 `child_process.spawn` 调用 `getLocalYaklangEngine()` 返回的仓库引擎。
- 输出：能力检查结果经 IPC 返回启动界面；成功后进入本地引擎启动流程。
- 环境：子进程继承当前环境并设置 `YAKIT_HOME`，不同产品版本可附加数据库环境变量。

### 六、技术选择与风险

- 将能力检查外层时限设为六十秒，为引擎内部十秒调用和 Windows 冷启动保留充足余量，同时仍能在异常进程长期无响应时终止任务。
- 不采用自动重试，避免重复创建能力检查进程及引入额外状态管理。
- 不处理现有样式兼容、依赖过期与钩子依赖警告，因为它们没有阻断开发服务器或 Electron 加载。
- 专用实时开发脚本跳过启动期类型检查，避免大型工程的检查进程占用数吉字节内存并阻塞 HTTP 就绪；生产构建和独立类型检查命令不受影响。
- 主要风险是失败反馈由十一秒延长至六十秒；该变化仅作用于启动前能力检查，且异常进程仍受时限控制。
- 跳过启动期类型检查意味着开发窗口不会即时报告类型错误，相关错误仍应通过独立静态检查命令发现。

### 七、验证策略

- 先增加会失败的源码契约断言，证明现有十一秒实现未满足验收条件。
- 修改后执行目标测试、语法检查、格式检查与差异检查。
- 使用 `start-enterprise-no-license-dev.cmd` 动态启动，检查最新引擎日志是否出现能力检查完成及本地引擎启动记录。
- 验证三千端口能够返回 HTTP 响应，且开发会话中不存在类型检查报告子进程。
