## 项目上下文摘要（客户端调试脚本）

生成时间：2026-07-19 22:47:57 +08:00

### 一、目标与范围

- `start-client-dev.cmd` 作为企业免许可证桌面客户端的唯一开发实现，负责环境检查、端口检查、长期监听提示和退出状态展示。
- `start-enterprise-no-license-dev.cmd` 作为兼容入口，只转调统一实现，避免两个批处理文件产生行为差异。
- 两个入口只调用既有客户端开发编排，不包含构建、打包、依赖安装或显式引擎命令。

### 二、相似实现分析

- `start-enterprise-no-license-dev.cmd`：采用 `setlocal`、脚本目录定位、开发环境变量、`call yarn` 与退出码传递，是批处理结构的直接依据。
- 根 `package.json` 的 `dev`：主渲染服务在端口 `3000` 可用后启动 Electron，体现开发服务与桌面外壳的就绪次序。
- 根 `package.json` 的 `dev-enterprise-no-license`：同时维持企业免许可证主渲染服务、企业启动页服务与 Electron，是本任务应复用的完整客户端编排。
- 根 `package.json` 的 `start-renders-enterprise`：证明两个渲染服务属于独立客户端进程，但该命令使用带许可证的主渲染配置，因此不能作为免许可证入口。

### 三、项目约定

- 批处理使用中文状态文本、双引号保护路径和变量、`cd /d "%~dp0"` 处理跨盘符路径，并以 UTF-8 无字节顺序标记及 Windows 换行保存。
- Node.js 当前版本为 `22.13.0`，Yarn 当前版本为 `1.22.22`。
- 主渲染端由 Webpack 开发服务器提供热更新；启动页由 Vite 与 React 插件提供热更新。

### 四、可复用能力

- `yarn dev-enterprise-no-license`：完整客户端开发生命周期管理。
- `REACT_APP_SKIP_DEV_TYPE_CHECK=true`：复用当前开发配置，移除开发期类型检查插件并关闭开发产物写盘。
- `DISABLE_ESLINT_PLUGIN=true`：当前 React Scripts 本地实现支持的开发期代码规范插件开关。
- `BROWSERSLIST_IGNORE_OLD_DATA=true`：当前 Browserslist 本地实现支持的过期数据提示开关。

### 五、依赖与集成点

- 根目录依赖 `concurrently`、`wait-on` 与 `electron`。
- 主渲染目录依赖 `env-cmd`、`react-app-rewired` 与 React Scripts。
- 启动页目录依赖 Vite 与 React 插件。
- `app/main/index.js` 在开发态固定读取 `http://127.0.0.1:3000` 与 `http://127.0.0.1:5173`，因此两个渲染服务都属于真实桌面客户端调试链。

### 六、日志判断

- 用户提供的日志显示主渲染端编译完成；Browserslist、Autoprefixer 与 ESLint 内容均为非阻断警告。
- 两个开发地址当前均返回 HTTP 成功状态，Electron 主进程及渲染进程均可响应。
- 应用日志显示两个窗口均完成页面加载，随后本地引擎能力检查等待六十秒并超时；资源目录检查函数已经返回，不是停滞位置。
- 相同引擎能力命令在相同用户目录与端口条件下复验约三秒完成，说明当前日志对应一次性执行异常或状态冲突，不能归因于前端编译。

### 七、测试策略

- 静态验证批处理必需变量、唯一 Yarn 调用、退出码传递、统一入口关系及未出现显式引擎命令。
- 使用根 `package.json` 解析确认复用命令仍包含两个渲染服务、双地址就绪等待与 Electron。
- 当前已有开发进程占用端口 `3000` 与 `5173`，以禁用暂停的验证环境变量执行两个入口，均显示占用说明并返回退出码 `2`，没有创建第二组客户端。
- 检查两个批处理文件均为 UTF-8 无字节顺序标记，且不存在单独换行字符。

### 八、风险与边界

- 命令窗口会持续存在，这是热更新服务维持监听所需的生命周期。
- 脚本本身不执行引擎命令，但启动页仍可能依据既有业务状态检查或启动本地引擎；完全禁止引擎需要改造应用启动协议，超出单脚本范围。
- 统一入口会在 `3000` 或 `5173` 已监听时停止启动并显示原因，避免 Vite 自动迁移端口后仍由 Electron 读取旧地址。
- 顺序思考、任务管理器、桌面文件管理器与 Context7 工具当前不可用；本次使用结构化计划、CodeGraph、定向源码检索、进程状态和本地依赖源码完成等价调查。
