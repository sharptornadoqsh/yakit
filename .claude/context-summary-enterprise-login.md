# 项目上下文摘要（企业登录界面）

生成时间：2026-07-14 11:31:09 +08:00

## 一、任务边界

- 工作分支为 `qsh`。
- 只调整企业登录页、共享登录表单的页面模式、登录阶段工具栏可见性和对应多语言文本。
- 保留私有域地址、用户名、密码、登录、窗口最小化、最大化与关闭操作。
- 保留登录失败后既有本地进入能力，但降低视觉层级；不修改触发条件。
- 不修改登录请求、密码摘要、许可证判断、配置缓存、数据同步、进程间通信或引擎逻辑。
- 工作区内已有阶段三记录和引擎生命周期测试，本任务不修改、不暂存这些文件。

## 二、相似实现分析

### 二点一、共享私有域表单

- 文件：`app/renderer/src/main/src/components/ConfigPrivateDomain/ConfigPrivateDomain.tsx`
- 模式：同一组件覆盖设置弹窗和企业登录，字段校验、配置保存、登录请求与成功回调集中在组件内。
- 可复用：`YakitAutoComplete`、`YakitInput`、`YakitButton`、既有表单校验与加载状态。
- 约束：企业登录页面需要独立纵向布局，设置弹窗必须保持原布局与行为。

### 二点二、企业许可证与登录门禁

- 文件：`app/renderer/src/main/src/pages/EnterpriseJudgeLogin.tsx`
- 模式：许可证状态决定显示激活页或企业登录表单，成功后退出登录门禁。
- 可复用：许可证判断、登录成功回调和免许可证构建分支。
- 约束：只能替换展示容器，不能改变许可证与登录状态机。

### 二点三、应用窗口控制

- 文件：`app/renderer/src/main/src/components/layout/UILayout.tsx`
- 文件：`app/renderer/src/main/src/components/layout/WinUIOp.tsx`
- 模式：应用工具栏与窗口控制共存，窗口控制通过既有桥执行最小化、最大化和关闭。
- 可复用：`WinUIOp` 与 `MacUIOp`。
- 约束：登录阶段隐藏首页、通知、工具、设置、帮助、性能和地址入口，但必须保留平台窗口控制。

### 二点四、旧登录与许可证页面

- 文件：`app/renderer/src/main/src/pages/Login.tsx`
- 文件：`app/renderer/src/main/src/pages/LicensePage.tsx`
- 模式：旧页面使用居中卡片和水平表单，功能路径完整但视觉层次有限。
- 可复用：弹窗企业登录仍使用共享表单的默认模式。
- 约束：页面模式改造不能影响企业登录弹窗和许可证激活页。

## 三、项目约定

- React 组件使用大驼峰，函数与变量使用小驼峰，样式采用相邻模块化样式或既有组件全局样式。
- TypeScript 与样式遵循两空格、单引号、无分号和尾逗号约定。
- 用户可见文本写入英文、简体中文和繁体中文资源，保持键集合一致。
- 复用现有主题变量、产品配置和原创品牌图标，不增加第三方依赖或新图片。

## 四、集成关系

```text
UILayout 登录门禁
  → EnterpriseJudgeLogin 专用页面外壳
    → ConfigPrivateDomain 页面模式
      → 既有配置保存与企业登录流程
      → 登录成功后 setJudgeLicense(false)
```

登录阶段工具栏只改变可见结构，不改变窗口控制桥。共享表单增加页面模式，默认模式继续服务设置弹窗和应用内企业登录弹窗。

## 五、测试策略

- 使用 Vitest 与 Testing Library 验证企业登录页面的品牌信息、正式表单容器和页面模式传递。
- 使用多语言键一致性检查验证新增文案。
- 使用 TypeScript、规则检查和企业免许可证主渲染构建验证集成。
- Electron 动态检查登录页只显示登录必要操作及窗口控制，并检查浅深主题。

## 六、风险

- `UILayout.tsx` 体量较大，只对登录门禁条件添加局部可见性判断。
- `ConfigPrivateDomain` 被多个入口复用，页面模式必须显式开启，默认表现不变。
- 工作区存在并行阶段三改动，验证与差异审查必须区分本任务文件。
