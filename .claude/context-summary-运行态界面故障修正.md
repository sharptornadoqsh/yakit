## 项目上下文摘要（运行态界面故障修正）

生成时间：2026-07-20 05:53:19 +08:00

### 一、目标与范围

- 修正报文差异页首次创建编辑器时的空模型销毁异常。
- 修正系统设置长表单区被弹性布局压缩后相互重叠的问题。
- 修正用户管理表格在多层分栏中自动列宽过小、账号字段难以辨认的问题。
- 保持现有接口、账号数据、组织树、角色逻辑、系统配置提交、路由和进程通信协议不变。

### 二、相似实现分析

- `YakitDiffEditor.tsx`：分别判断编辑器及左右模型后执行生命周期清理，不依赖尚未绑定的模型。
- `YakitMonacoDiffInline.tsx`：创建模型后立即绑定，并在卸载时使用保存的模型引用，证明左右模型需要独立判空。
- `react-monaco-editor` 的差异编辑器实现：原模型与修改模型分别判断后销毁，与当前 Monaco 版本的空模型行为一致。
- `TrustListPage.tsx`、`RoleAdminPage.tsx` 与 `LicenseAdminPage.tsx`：窄表格或含头像字段使用明确列宽，操作列保留固定宽度。
- `RuiYanFormSection`：公共组件面向固定高面板使用弹性正文；系统设置页面属于长文档流，需要页面级覆盖其高度策略。

### 三、直接证据

- Monaco `0.40.0` 的差异编辑器在首次绑定前返回 `{ original: null, modified: null }`；原代码只判断外层对象便调用 `dispose()`，异常文本与截图一致。
- 用户管理截图已有一条真实记录；中间表格受外层百分之七十二分栏、内部组织树最小宽度和固定操作列共同压缩，四个自动列的可用空间不足。
- 系统设置运行态截图显示多个表单区正文占用接近零高度，内容绘制到相邻区块；公共表单正文的弹性规则与页面长内容模式存在冲突。

### 四、实现约束

- 报文差异仅对子模型增加独立判空，并在销毁编辑器前清除引用。
- 用户管理只为四个信息列设置明确宽度，保留原操作列宽度、横向滚动、行选择和详情事件。
- 系统设置只提高页面选择器优先级，使设置区和正文采用普通文档流高度，不修改公共表单组件。

### 五、验证策略与当前边界

- 两个 TypeScript 页面定向 ESLint 为零错误、十项既有警告。
- 目标差异通过 `git diff --check`。
- 现有开发服务确认已读取初版样式；初版运行态复查仍有重叠，因此改为更明确的块级文档流规则。
- 第二版样式写入后，客户端窗口处于最小化状态，未强制恢复窗口；最终视觉结果留待用户现有开发会话或后续编译检查。
- 依照用户指令，没有执行渲染器构建、类型编译、打包或第二组开发服务启动。
