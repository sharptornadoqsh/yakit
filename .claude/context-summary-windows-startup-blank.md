## 项目上下文摘要（Windows 启动空白与标题栏平台识别）

生成时间：2026-07-15 16:44:58 +08:00

### 1. 相似实现分析

- `app/main/index.js` 的 `engineLinkWin-done`：启动窗口完成引擎初始化后隐藏自身、显示主窗口并发送认证信息，是初次启动的既有窗口切换协议。
- `app/main/index.js` 的 `yakitMainWin-done`：主窗口切换引擎时隐藏自身、显示启动窗口并发送当前信息，证明两个窗口通过完成事件互斥显示。
- `app/renderer/engine-link-startup/src/pages/StartupPage/index.tsx`：工作区确认状态初始为假，确认后才调用引擎初始化入口，说明该页面属于必要交互，不属于可直接隐藏的装饰画面。
- `app/renderer/src/main/src/components/BaseTitleBar/index.tsx`：组件挂载后独立读取系统类型，再选择苹果系统或其他系统按钮，提供了标题栏平台初始化模式。
- `app/renderer/src/main/src/components/baseConsole/BaseConsole.tsx`：组件挂载后读取系统类型并保存，证明系统查询不需要等待引擎连接完成。

### 2. 项目约定

- 主进程窗口生命周期位于 `app/main/index.js`，窗口显示统一使用 `winShow`，隐藏统一使用 `winHide`。
- 主界面组件使用函数组件、挂载副作用和已有桥接接口；命名采用小驼峰，缩进为两个空格，单引号且不使用分号。
- 测试位于相邻 `__test__` 目录；当前产品测试通过读取源文件验证构建与窗口契约，本任务沿用该测试结构。

### 3. 可复用组件清单

- `app/main/index.js`：`winShow`、`winHide`、`engineLinkWin-done` 与 `yakitMainWin-done`。
- `app/renderer/src/main/src/constants/hardware.ts`：`handleFetchSystem` 与 `SystemInfo`。
- `app/renderer/src/main/src/services/electronBridge.ts`：现有系统信息桥接接口。
- `app/main/__test__/product.test.js`：产品启动契约测试。

### 4. 测试策略

- 使用现有产品测试复现并约束：启动窗口可见、主窗口在资源加载完成时保持隐藏、完成事件负责切换窗口。
- 约束主界面在挂载阶段独立读取系统类型，避免标题栏平台判断依赖引擎事件。
- 执行产品定向测试、主渲染器类型检查、企业免许可证双渲染构建和差异检查。
- 使用 Windows 桌面控制观察工作区确认界面；完成确认后观察主界面非空白及按钮位于右上角。

### 5. 依赖和集成点

- 输入：Electron 两个渲染窗口的就绪事件、工作区确认、引擎完成事件、系统类型查询结果。
- 输出：启动窗口与主窗口的互斥可见状态、主界面初始化数据、平台对应的标题栏按钮。
- 配置：企业免许可证渲染命令仍为 `yarn build-renders-enterprise-no-license`，本任务不改变类别或许可证逻辑。

### 6. 技术选型理由

- 恢复既有双窗口协议，可以保留强制工作区确认与引擎状态展示，不需要建立新的启动状态管理机制。
- 平台查询沿用已有工具函数，修改局限于主界面初始化阶段，避免绑定到引擎认证事件。
- 不改动工作区组件、引擎协议或许可证流程，降低对其他产品类别的影响。

### 7. 关键风险点

- 启动窗口若继续隐藏，工作区确认无法发生，引擎完成事件永久缺失。
- 主窗口若在完成事件之前显示，所有主体渲染条件均为假，会再次出现持续空白。
- 平台查询若仍依赖完成事件，异常或延迟状态下 Windows 会继续显示苹果系统按钮。
- 当前测试曾将错误的直接显示行为作为期望值，必须同步修正测试契约。
