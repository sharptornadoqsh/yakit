# 项目上下文摘要（睿眼二三级页面统一风格）

生成时间：2026-07-17 10:56:51 +08:00

## 一、目标与验收条件

本任务继续执行睿眼前端整体重构。全部一级入口下保留的二级、三级页面需要采用五张效果图一致的视觉和交互体系；需求工作簿未覆盖、没有公共业务职责且没有真实操作链路的界面元素可以隐藏或删除。既有路由、属性、状态、事件、接口、进程间通信、远程过程调用与引擎调用保持不变。

验收条件如下：

- 十个一级入口继续使用固定深色全局框架、顶部命令栏和当前模块二级目录。
- 保留页面采用统一页面头、工具栏、卡片、分栏、表格、表单、状态、弹窗和抽屉结构。
- 二级、三级目录不得显示没有操作结果的禁用入口；需求相关能力必须仍能从真实页面访问。
- 页面不得重复显示全局返回入口，不恢复旧的多层可关闭页签结构。
- 主要浮层使用睿眼独立结构；迁移时继续调用原提交函数和数据接口。
- 工作簿星号需求全部保持，非星号十二项既定范围保持。
- 不修改用户保留的引擎、开发配置和启动脚本改动。
- 不执行构建、打包、暂存、提交、推送、合并请求和远程流水线。

## 三、相似实现分析

- `app/renderer/src/main/src/components/renyanUI/RuiYanShell.tsx`
  - 模式：独立顶部命令栏、一级导航与二级导航，真实路由继续交给既有页面打开回调。
  - 可复用：`RuiYanAppShell`、`RuiYanTopCommandBar`、`RuiYanPrimaryNav`、`RuiYanSecondaryNav`。
  - 约束：当前二级目录仍显示无路由节点，并以禁用状态和“页内”标记呈现。
- `app/renderer/src/main/src/components/renyanUI/RuiYanPrimitives.tsx`
  - 模式：原生语义元素与独立样式模块实现按钮、表格、页签、状态、浮层、表单和详情面板。
  - 可复用：全部睿眼公共原语及其焦点、读取、禁用和键盘交互。
  - 约束：弹窗与抽屉缺少副标题、分区导航和内容容器扩展接口，部分页面仍直接使用旧浮层。
- `app/renderer/src/main/src/pages/loginOperationMenu/RoleAdminPage.tsx`
  - 模式：角色列表继续使用真实服务数据，选中项通过角色详情接口读取权限内容，右侧面板不构造服务未返回的数据。
  - 可复用：列表查询、选中状态、角色表单、权限详情读取、创建和删除事件。
  - 约束：角色编辑仍使用旧模态框，权限矩阵主要呈现标签，和效果图中的分组矩阵存在视觉差距。
- `app/renderer/src/main/src/pages/loginOperationMenu/AccountAdminPage.tsx`
  - 模式：组织、用户列表、用户详情三栏使用真实用户和部门接口。
  - 可复用：组织查询、用户查询、详情状态、新建编辑和密码重置事件。
  - 约束：部分操作浮层和表格仍沿用旧结构。
- `app/renderer/src/main/src/pages/hTTPHistoryAnalysis/HTTPHistoryAnalysis.tsx`
  - 模式：筛选条件、流量表格、请求响应详情由原查询与页面状态组成。
  - 可复用：查询参数、分页、筛选器、流量详情和发送至重放事件。
  - 约束：页面需要由统一视觉容器控制表格密度、详情边界和工具栏层级。

## 四、项目约定

- 组件与类型采用大驼峰；函数、变量和属性采用小驼峰；钩子以 `use` 开头。
- TypeScript、React 和 Sass 使用两个空格、单引号、无分号、尾随逗号和一百二十字符宽度。
- 测试位于相邻 `__test__` 目录，使用 Vitest、Testing Library 和 `jsdom`。
- 用户文案使用简体中文，代码标识符保持英文。
- 显示层通过 `YakitRoute`、`openMultipleMenuPage`、事件总线和原页面组件集成，不新增业务协议。

## 五、可复用组件与集成点

- `routes/renyanMenu.ts`：菜单模型、能力过滤、功能标志、显示名称和活动路径。
- `components/renyanUI/RuiYanVisualContext.tsx`：路由对应的布局语义和状态资源。
- `components/renyanUI/RuiYanPrimitives.tsx`：公共控件、内容容器和浮层。
- `pages/layout/mainOperatorContent/MainOperatorContent.tsx`：页面缓存、路由实例与原业务页面挂载点。
- `components/layout/RenyanPageHeader.tsx`：统一页面标题和模块说明。
- `pages/layout/renyanMenu/RenyanNavigation.tsx`：菜单选择到原路由和设置动作的适配。

## 六、测试策略

- 菜单测试验证十个一级入口、可见二级入口、隐藏元素和真实路由值。
- 导航测试验证活动路由同步、搜索、一级入口和二级入口的真实选择回调。
- 公共组件测试验证按钮、页签、表格、模态框、抽屉和键盘行为。
- 页面定向测试验证迁移浮层仍调用原事件，不改变请求参数。
- 最终执行修改文件相关测试、格式、类型、样式解析、导入路径、差异空白和工作区状态检查。

## 七、依赖与风险

- 同一路由包含多个业务区域，二级目录不能通过重复路由制造无法区分的活动状态。
- 团队与角色页面依赖外部服务，界面只能呈现真实返回值和真实空状态。
- 全局样式必须限制在睿眼页面根节点，避免影响登录页、启动页与旧产品类别。
- 当前工作区已有四个修改文件和一个未跟踪文件，均作为用户保留内容，不纳入本任务。
- 工作区缺少顺序思考、任务管理器、本地文件管理器、编程文档和远程代码搜索接口；本任务使用会话计划、代码图、项目搜索、表格只读接口和本地源码证据完成对应分析。

## 八、技术选择

现有睿眼框架和公共组件已经覆盖主要基础能力，应在其上扩展页面布局语义、浮层结构和路由覆盖。二级目录中没有真实路由或动作的项目依据需求矩阵处理：需求外项目隐藏；需求内项目由所属真实业务页面承载，不显示没有操作结果的按钮。该选择避免伪造页面，同时保持业务入口清晰。

## 九、实施结果

- 保留的睿眼页面统一由页面根节点限定视觉规则，覆盖页签、卡片、表单、表格、空状态、深色作业区和分栏边界；登录页、启动页与其他产品类别不受该规则影响。
- 二级目录只显示具有真实路由、真实动作或可导航后代的节点；没有操作结果的规划节点和需求外元素不再呈现。
- 全局可关闭页签带在睿眼页面中隐藏，页面头移除重复返回入口，并以真实菜单路径生成面包屑。
- 公共组件增加面包屑、分段控制、权限矩阵、代码编辑框架和命令式模态框；模态框与抽屉增加副标题、分区导航、焦点约束和退出键处理。
- 用户与角色、代理、重放、网络配置、编解码、插件与爆破页面的主要浮层改为睿眼公共结构；原表单状态、提交处理、接口、进程间通信、远程过程调用及引擎调用保持。
- 工作簿十四项星号要求全部保留，十七项非星号要求覆盖十二项，比例为百分之七十点六；需求映射继续以 `.claude/renyan-feature-map.md` 为准。
- 五张效果图只用于视觉结构提取，本次没有引入外部图片；界面继续使用仓库内的睿眼矢量图、状态资源和图标体系。
