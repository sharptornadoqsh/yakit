# 项目上下文摘要（睿眼登录后整体重构）

生成时间：2026-07-20

## 一、目标、范围与验收条件

本任务依据指定目录内全部四十个效果图文件，重建登录后的应用框架、公共组件和真实业务页面。用户已允许整体替换此前的睿眼展示实现。企业登录、用户管理、角色管理、组织管理、权限管理、引擎、协议、后端、数据库和生成目录不产生新的修改。页面继续调用现有 Hook、事件、网络接口、进程间通信和远程过程调用，不使用模拟业务数据。

验收以页面映射完整、统一深色导航与白色工具栏成立、底部状态栏不遮挡内容、任务中心从底部展开、消息面板从右上展开、真实业务功能可调用、受保护文件无新增差异、渲染器检查通过且未执行安装包构建为准。

## 二、效果图与页面映射

指定目录共四十个 PNG 文件，哈希比对确认附件图片与目录文件一致。三组文件内容重复，附件编号分别为十七与十八、二十一与二十二、三十五与三十六，因此唯一画面为三十七张。完整业务映射见 `docs/ui-redesign/ui-page-mapping.md`。

视觉方向采用传统政企安全工作台：深蓝灰导航、白色顶部工具栏、浅灰工作区、细边框白色面板、深蓝主操作、较小圆角、高密度表格、固定底部状态栏。实现遵循前端设计技能的明确视觉方向，同时服从效果图关于无大面积渐变、无玻璃效果、少投影和克制动效的约束。

## 三、相似实现分析

### 实现一：应用外壳和导航

- 文件：`components/renyanUI/RuiYanShell.tsx`、`pages/layout/renyanMenu/RenyanNavigation.tsx`。
- 模式：菜单模型与视图分离，导航动作最终复用原路由或事件。
- 可复用：`RuiYanAppShell`、`RuiYanTopCommandBar`、`RuiYanPrimaryNav`、`RuiYanSecondaryNav`。
- 风险：此前一级导航仍为浅色窄栏，整体宽度和目标深色侧栏存在差异。

### 实现二：公共原语和浮层

- 文件：`components/renyanUI/RuiYanPrimitives.tsx`、`RuiYanUI.module.scss`。
- 模式：语义组件通过样式模块共享设计变量，门户浮层统一处理焦点和退出键。
- 可复用：按钮、面板、数据表、面包屑、页签、状态、弹窗、抽屉和确认框。
- 风险：缺少专用下拉、提示、通知和消息原语，旧封装需要主题适配。

### 实现三：页面视觉作用域

- 文件：`components/renyanUI/RuiYanVisualContext.tsx`、`pages/layout/mainOperatorContent/MainOperatorContent.tsx`、`routes/newRoute.tsx`。
- 模式：路由保持不变，通过视觉映射决定页面头、空状态和布局类型。
- 可复用：`routeVisuals`、`RuiYanVisualProvider`、`RenyanPageHeader`、页面级错误边界。
- 风险：此前只覆盖十七类路由，报告、端口、快捷键和部分真实页面尚未纳入。

### 实现四：消息中心和任务抽屉

- 文件：`components/MessageCenter/MessageCenter.tsx`、扫描任务抽屉和任务 Hook。
- 模式：真实消息与任务状态由现有 Hook、事件和接口提供，视图容器可以替换。
- 可复用：消息查询、已读、清理、套接字刷新、任务导入和已有任务表格。
- 风险：此前没有正式全局任务中心，不能把特定扫描任务或静态数组冒充全部任务。

## 四、项目约定

- 技术栈：React、TypeScript、Sass 模块、Vitest、Electron。
- 命名：组件和类型使用大驼峰，函数与变量使用小驼峰，Hook 以 `use` 开头。
- 风格：两个空格、单引号、无分号、尾随逗号、一百二十字符宽度。
- 集成：页面经 `YakitRoute` 和 `RouteToPage` 挂载；壳层动作经事件总线与 `UILayout` 处理。
- 测试：相邻 `__test__` 目录，覆盖正常路径、边界、失败恢复和键盘行为。

## 五、可复用组件与依赖

- `routes/renyanMenu.ts`：菜单、能力、功能标志与路由标签。
- `components/renyanUI`：统一展示原语和视觉上下文。
- `components/layout/RenyanStatusBar.tsx`：真实引擎、项目和团队状态。
- `components/MessageCenter`：真实消息与任务通知。
- `HybridScanExecuteContent`：扫描向导和策略参数。
- `HTTPFlowTable` 与详情组件：流量列表、筛选和报文详情。
- `ReportViewerPage` 与 `CreateReportContent`：报告中心、预览和生成。
