# 插件开发取消退出实施计划

生成时间：2026-07-21 17:12:04 +08:00

## 范围

- 修改全页插件开发组件、主页面关闭请求协议、监听逻辑与确认框生命周期控制。
- 新增相邻回归测试与本任务审查记录。
- 抽屉编辑交互、保存协议、插件数据结构与翻译资源保持不变；仅显式关闭其全页订阅能力。

## 执行步骤

1. 新增组件测试，验证全页取消入口及关闭请求；当前代码应因入口缺失而失败。
2. 为主页面事件协议增加确认关闭请求，并由页面容器转交现有关闭检查。
3. 在全页插件开发态注入响应式取消按钮，将头部插槽移至操作组最左侧。
4. 隔离全页与抽屉编辑器的关闭订阅所有权，并让快捷键状态跟随确认框真实销毁。
5. 执行定向测试、格式检查、代码规范检查、类型检查、多语言检查和差异检查。
6. 独立复查交互范围、内容保护和工作树差异，生成验证报告。

## 回退方式

删除新增事件、监听、取消按钮、订阅开关、确认框生命周期辅助模块和相邻测试，并将 `headerExtra` 恢复到原位置；其余插件编辑与保存实现不受影响。
