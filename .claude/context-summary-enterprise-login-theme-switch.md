# 项目上下文摘要（企业登录主题选择）

生成时间：2026-07-15 22:55:00 +08:00

## 一、任务边界

- 工作分支为 `qsh`，开始时工作树无改动。
- 保留现有双栏企业登录页、品牌区、服务地址、用户名、密码、登录按钮和窗口控制。
- 在登录页增加浅色与深色主题选项，选择结果沿用应用现有主题状态并持久化。
- 不修改认证请求、密码摘要、许可证判断、远端配置、账号缓存、数据同步、进程间通信或引擎逻辑。

## 二、相似实现分析

### 二点一、应用主题状态

- 文件：`app/renderer/src/main/src/hook/useTheme/index.tsx`。
- 模式：`theme` 提供当前值，`setTheme` 同步文档主题属性、本地存储、主题变量和主进程。
- 可复用：`Theme` 类型、`theme` 与 `setTheme`，不新增主题状态。

### 二点二、设置菜单主题切换

- 文件：`app/renderer/src/main/src/components/layout/FuncDomain.tsx`。
- 模式：浅色和深色操作直接调用现有 `setTheme`。
- 可复用：现有主题值与切换协议，登录页选择后与工作台设置保持一致。

### 二点三、按钮式单选组件

- 文件：`app/renderer/src/main/src/components/yakitUI/YakitRadioButtons/YakitRadioButtons.tsx`。
- 模式：基于单选组提供受控值、选项与变更事件，样式使用全局主题变量。
- 可复用：小尺寸实心按钮模式，满足两个互斥主题选项。

### 二点四、企业登录页

- 文件：`app/renderer/src/main/src/pages/EnterpriseJudgeLogin.tsx` 与相邻样式、测试。
- 模式：右侧表单顶部已有访问入口和环境标记，适合加入紧凑主题选择；认证表单保持独立组件。
- 可复用：现有表单顶部区域、模块化样式和相邻组件测试。

## 三、项目约定

- React 组件使用大驼峰，函数与变量使用小驼峰，样式采用相邻模块化样式。
- 代码使用两个空格、单引号、无分号与尾随逗号。
- 用户可见文本同步维护英文、简体中文和繁体中文资源。
- 图标复用现有太阳与月亮轮廓图标，不新增图片或第三方依赖。

## 四、依赖与集成点

```text
EnterpriseJudgeLogin
  → useTheme 当前主题与切换函数
  → YakitRadioButtons 浅色或深色互斥选择
  → ConfigPrivateDomain 既有认证表单
```

## 五、测试策略

- 相邻组件测试验证当前主题选中状态、浅色与深色文案、切换调用和认证表单参数。
- 运行多语言键一致性、TypeScript、格式与企业免许可证双渲染器生产构建。
- 使用桌面界面查看主题选择位置、两种主题切换效果和登录表单完整性。

## 六、风险与边界

- 主题选择会持久化并影响登录后的工作台，这是现有设置菜单的相同行为。
- 主题按钮位于右侧表单顶部，窄窗口由现有单栏响应式布局承载。
- 不在桌面验证中提交登录表单或记录用户凭据。

## 七、上下文充分性检查

- 接口契约明确：受控主题值只接受 `light` 与 `dark`。
- 技术选择明确：复用现有主题状态与按钮式单选组件。
- 风险已识别：持久化、窄屏布局与多语言宽度均有验证方式。
- 验证方式明确：组件测试、多语言检查、类型检查、生产构建与桌面查看。
