# 项目上下文摘要（精简桌面标题栏）

生成时间：2026-07-17 00:56:15 +08:00

## 相似实现分析

- `components/layout/UILayout.tsx` 的微软系统分支同时渲染全局状态、模式入口、功能域、帮助、性能采样、引擎信息与 `WinUIOp`。截图中的冗余内容来自该分支。
- `components/layout/UILayout.tsx` 的苹果系统分支采用相反的窗口控制位置，并复用 `MacUIOp`、功能域、性能展示和全局状态。
- `components/renyanUI/RuiYanShell.tsx` 已提供睿眼品牌、搜索、任务、环境、通知和用户入口；旧标题工具区继续存在会形成重复导航。
- `components/layout/WinUIOp.tsx` 与 `components/layout/MacUIOp.tsx` 封装原窗口控制、最大化状态和关闭前确认，适合直接复用。

## 项目约定

- 组件与类型使用大驼峰，函数与变量使用小驼峰。
- React、TypeScript 与 Sass 模块采用两个空格、单引号、无分号和尾随逗号。
- 测试使用 Vitest、Testing Library 和模块模拟，放置于相邻 `__test__` 目录。

## 依赖与集成点

- 睿眼启用状态来自 `routes/renyanMenu.ts` 的 `RENYAN_SHELL_ENABLED`。
- 标题栏容器由 `UILayout` 管理，其下方是主应用与登录等页面。
- 拖动区域继续使用 `header-title-drop`，双击行为继续调用 `maxScreen`。
- 窗口最小化、最大化、关闭和关闭前确认继续由 `WinUIOp` 与 `MacUIOp` 负责。

## 测试策略

- 独立渲染极简窗口框，并模拟两种平台窗口控制组件。
- 验证微软系统只展示微软系统窗口控制，苹果系统只展示苹果系统窗口控制。
- 验证拖动区域和双击回调存在，不引入旧工具区内容。
- 对改动文件执行定向 Vitest、TypeScript 语法转换和 Sass 解析。

## 风险与边界

- 不删除旧工具组件，非睿眼分支继续保留旧标题栏结构。
- 不改变窗口关闭确认、引擎状态、状态存储或事件协议。
- 不启动桌面应用，不执行构建或全量检查；视觉效果由用户手动查看。

## 验收条件

- 睿眼模式的最上层仅保留拖动区域和平台窗口控制。
- 旧全局状态、模式、功能域、帮助、性能和引擎信息不再出现在该区域。
- 最小化、最大化、关闭、拖动与双击最大化能力保持。
