# 项目工作区极简布局实施计划

> 执行要求：按任务逐项实施并验证，使用复选框记录状态。

**目标：** 在保留全部项目管理信息与行为的前提下实现紧凑双栏工作区。

**架构：** 继续使用 `ProjectManage` 的现有状态、事件和操作函数，仅增加操作按钮的语义分组，并在对应样式模块内调整网格、间距、边界和响应式规则。

**技术栈：** React、TypeScript、样式模块、既有睿眼组件与主题变量。

## 全局约束

- 不新增运行时依赖。
- 不改项目管理接口和数据结构。
- 不执行安装包构建。
- 所有验证在本地完成。

### 任务一：操作区语义分组

**文件：**

- 修改：`app/renderer/src/main/src/pages/softwareSettings/ProjectManage.tsx`

**接口：**

- 使用现有 `operateFunc` 与 `setProjectShareModal`。
- 不产生新的业务接口，仅增加本地操作组和团队协作组容器。

- [ ] 将临时项目、新建项目、新建文件夹和导入归入本地操作组。
- [ ] 将分享当前团队项目和通过密令导入归入团队协作组。
- [ ] 保持按钮文字、图标、点击处理和顺序不变。

**接口：**

- 为任务一的两个容器提供布局规则。

- [ ] 缩减工具栏、侧栏和内容区的间距。
