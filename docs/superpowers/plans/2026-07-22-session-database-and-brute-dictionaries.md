# 会话、数据库与爆破字典实施计划

## 目标

在不改变现有服务端协议的前提下，修复数据库关闭后的业务查询、企业会话不同步、令牌缺少持续刷新、并发过期通知重复以及弱口令字典下拉被遮挡的问题。

## 实施步骤

一、增加失败回归测试。

- 公共抽屉内的 Ant Design 下拉选项必须挂载到当前对话框。
- 主进程已有登录态必须同步到渲染端；登录后立即刷新，并按十分钟周期刷新。
- 同一令牌连续触发过期处理时只退出和提示一次。

二、统一会话生命周期。

- 新增单一会话钩子，查询主进程用户状态并订阅登录事件。
- 登录状态有效时调用现有 `refreshToken`，监听既有 `refresh-token` 通道并建立定时刷新。
- 在 `MainOperator` 中替换分散且捕获陈旧状态的三个副作用。

三、限制过期处理次数。

- 以响应携带的令牌识别会话。
- 同一会话只执行一次本地退出、动态控制退出、全局令牌清理与通知。
- 新登录同步时重置过期标记，使后续真实失效仍可处理。

四、保护项目数据库生命周期。

- 项目管理弹窗不允许通过关闭按钮或退出键离开。
- 仅允许 `ProjectManage` 在成功选定项目后触发完成回调并重新展示业务页面。

五、修复抽屉内下拉层级。

- 公共浮层通过 Ant Design 配置将内部弹出节点挂载到浮层根节点。
- 字典顶层选项显式声明名称值，保持与 `Codec` 的字典名称协议一致。

六、本地验证。

- 执行新增及相关测试。
- 执行渲染端类型检查、目标代码规范检查、国际化检查与差异空白检查。
- 同步代码图并根据受影响测试列表复查。
- 在 `.claude/verification-report.md` 记录技术评分、需求评分、风险与结论。

## 验收标准

- 字典管理存在的顶层字典与文件夹内字典均能在弱口令抽屉中显示和选择。
- 应用已有登录态时，用户管理与角色管理不再显示重新登录状态。
- 登录期间刷新接口会即时执行且每十分钟再次执行；现有窗口刷新事件使用最新用户状态。
- 同一令牌的并发四百零一响应只产生一次退出与一次提示。
- 项目管理未选定项目时无法关闭弹窗，报告与流量页面不会在数据库关闭期间重新挂载。
- 所有定向测试、类型检查和目标静态检查成功。
