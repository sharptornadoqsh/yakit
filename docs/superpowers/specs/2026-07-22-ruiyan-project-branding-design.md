# 睿眼项目与品牌外观设计说明

## 目标与范围

本次变更处理四项外部可见内容：项目入口确认文案、项目管理弹层视觉、默认数据库名称与默认项目详情、产品图标及名称与关于页面和安装身份。内部类型名、缓存键、接口名、引擎技术名与通信协议保持不变。

## 方案比较

### 采用方案：集中配置与无损迁移

在 `product/renyan.json` 增加社区版和企业版默认数据库名。主进程在本地引擎启动前，把旧数据库及其 SQLite 附属文件改为新名称；目标文件已经存在时，旧文件改为无旧品牌的归档名称。项目管理界面对默认项目字段增加展示规范化，以处理迁移前的服务缓存或旧描述。

### 未采用方案：仅替换界面文字

该方案改动范围较小，但磁盘数据库文件仍为旧名称，无法满足截图三的要求。

### 未采用方案：全仓品牌符号改名

该方案会影响内部类型、环境变量、缓存协议与引擎兼容字段，超出用户限定的外部特征范围。

## 界面设计

项目入口确认框标题改为“进入睿眼项目工作区”，正文使用一个完整段落，说明工作区用途与当前任务停止行为。

项目管理弹层采用紧凑、扁平、低装饰的极简方向：移除内层重复标题和卡片阴影；总数与搜索合并为单行工具栏；分组栏缩窄并使用分隔线；四个创建与导入操作改为紧凑按钮；表格使用轻量表头、细分隔线与低对比悬停状态；当前项目信息保留在分组栏底部。

## 数据与迁移

社区版数据库名为 `default-RuiYan.db`，企业版数据库名为 `company-default-RuiYan.db`。迁移同时处理空后缀、`-wal` 和 `-shm` 三类文件。迁移只执行同目录改名，不覆盖、不删除文件。

默认项目的描述和存储路径在展示时把旧品牌片段替换为 `RuiYan`。该规则不作用于普通项目，避免改写用户数据。复制路径仍使用服务返回的真实路径；引擎重启完成迁移后，真实路径与展示路径一致。

## 产品外观复查

应用图标继续使用 `app/assets/renyan-icon.*`，字标继续使用睿眼字标资源。应用名称、可执行文件、用户数据目录和安装产物继续由 `product/renyan.json` 与打包配置提供。关于页面继续从同一产品配置读取名称、字标、版本、支持方和版权信息。

## 验收标准

- 项目入口确认框只含一个睿眼相关正文段落，并保留任务停止提示。
- 项目管理弹层无重复标题、无大面积彩色操作卡片，窄屏下仍可使用。
- 新数据库及迁移后的旧数据库文件名不含旧品牌，且包含 `RuiYan`。
- 默认项目列表和详情抽屉不显示旧品牌；普通项目内容不被改写。
- 图标、应用名称、关于页面、可执行文件、用户数据目录与安装产物均使用睿眼身份。
- 定向测试、类型检查、主渲染构建和品牌残留扫描成功。
