# 项目工作区极简布局设计

## 目标

在不改变项目管理业务行为的前提下，将项目工作区调整为紧凑、克制的数据管理界面。保留项目数量、搜索、类型筛选、当前项目、项目创建、文件夹创建、本地导入、团队项目分享、密令导入、路径导航、列表字段与行操作。

## 设计方向

采用紧凑双栏结构。左侧保留项目类型导航和当前项目信息，右侧承担操作区、路径与项目列表。视觉层级以中性色和细边框为主，主色仅用于当前筛选、新建项目和必要的交互反馈。

## 布局规则

- 顶部统计与搜索保持同一水平线，缩短垂直占用。
- 左侧导航宽度缩小，当前项目卡片紧邻筛选区域，不再固定在页面底部。
- 操作区分为本地项目操作与团队协作操作，两组在宽屏下分居两侧，在窄屏下自然换行。
- 表格增加完整边界和轻量圆角，表头、数据行与选中状态采用低对比层级。
- 项目路径、复制入口、操作图标和空状态继续沿用现有交互。

## 约束

- 不修改数据请求、项目切换、增删改、导入导出和团队分享处理函数。
- 不新增依赖，不改变既有组件体系和主题变量。
- 支持现有浅色、深色及产品变体主题。
- 保持键盘焦点和按钮语义。

## 验收标准

- 所有原有信息和操作入口仍可访问。
- 宽屏下操作区不拥挤，团队操作形成独立视觉分组。
- 窄屏下侧栏与操作区不会遮挡表格。
- 完整类型检查无错误，相关单元测试通过。
