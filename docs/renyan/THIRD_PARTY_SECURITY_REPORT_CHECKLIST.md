# 第三方安全检测材料清单

本文件仅为第三方检测材料清单，不是第三方安全检测报告，不代表已经通过检测。

## 二、上线前必需材料

- 检测合同或任务书，明确产品版本、部署范围、互联网暴露面和责任主体。
- 检测机构资质、报告原件、签章页、报告编号与检测日期。
- 受测客户端、外部账号服务、域名、接口、端口、证书和配置清单。
- 登录、令牌、用户权限、角色权限、密码复杂度、密码存储和日志敏感信息测试记录。
- TLS 协议、证书链、弱协议、弱套件、HTTP 跳转和自签名证书处理记录。
- 产品本身及第三方组件的公开漏洞识别结果、适用性判断和版本依据。
- 每个问题的严重程度、复现条件、影响范围、处置措施和复验结果。
- 未处理问题的例外批准、缓解措施、有效期和责任人。

## 三、账号服务专项检查

- 验证 MD5 值只能经 HTTPS 传输，并记录其可重放风险。
- 验证原始密码与 MD5 值不进入客户端配置、日志和错误报告。
- 验证服务端密码存储算法、盐值、参数、数据库字段和历史迁移。
- 验证令牌签发、有效期、撤销、轮换、退出和服务地址隔离。
- 使用管理员、普通用户、审核员或其他实际角色直接请求用户与角色接口。
- 绕过前端提交弱密码，确认服务端仍拒绝不符合策略的输入。
- 验证 `Authorization`、Cookie、令牌、密码、私钥和远程引擎密码的日志处理。

## 四、材料接受条件

报告必须对应实际交付版本和实际部署边界。问题处置必须提供复验记录，不能只提供文字承诺。面向互联网部署前，应确认报告仍在约定有效期内，受测配置与上线配置一致，未处理高风险问题具有正式例外决定。
