## 证据状态

| 状态 | 判定含义 |
| --- | --- |
| 充分静态证据 | 当前源码包含覆盖需求主要行为的入口、数据结构及调用路径，仍需真实环境验收 |
| 部分静态证据 | 当前源码只覆盖需求的一部分，或关键服务端行为未包含在仓库中 |
| 未发现直接证据 | 在已记录调查范围内未定位到可支撑该行为的直接实现 |
| 待动态验证 | 需求依赖并发、时长、资源或多用户运行数据，静态源码无法给出结论 |
| 待外部材料 | 需求依赖供应商报告、外部规范或漏洞处置材料 |
| 存在冲突风险 | 当前源码出现可能与需求目标不一致的实现线索，需专项验证和处置 |

这些状态只表示基线证据强度，不代表采购验收、生产环境验证或合规结论。
