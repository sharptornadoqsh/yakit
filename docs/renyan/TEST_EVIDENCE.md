- 一次 Windows 诊断打包因临时目录文件占用返回 `EPERM`；核对目标后只清理 `release/win-unpacked.tmp`，重试成功。
- 用户调整验证目标时，正在执行的社区版生产构建被主动终止，随后改为普通企业版开发构建，最终再改为企业版免许可证开发构建。

1. 在仓库根目录设置 `ELECTRON_IS_DEV=0` 并执行 `yarn start-electron`，从企业入口选择“跳过”后进入本地项目。
2. 分别选择浅色和暗色主题，检查工作台标题、蓝色主色、青绿色连接状态、真实空状态、快捷入口及底部状态区。

4. 确认团队工作区与成员角色显示“待团队阶段实现”且不可导航；确认国密算法、插件链路编排、报告导出、客户端测试总览和序号二十五菜单均不可见。
5. 从流量分析进入报文重放，确认窗口标题为“报文重放 · 睿眼自动化渗透系统”；从系统管理打开关于对话框，确认产品、客户端、引擎、构建及“企业版（免许可证）”类别。
6. 需要扩展视觉证据时，将浅色工作台、展开菜单和关于对话框分别保存到 `docs/renyan/evidence/stage2`，不得改写统计数据或界面状态。
