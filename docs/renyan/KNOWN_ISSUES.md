# 睿眼产品化已知事项

## 阶段一：产品身份与构建适配

## 一、有界检索方法

最终源码使用以下有界检索统计大小写不敏感的 `Yakit` 与 `Yaklang` 字符串：

```text
rg --count-matches -i 'yakit|yaklang' -- <指定范围>
```

| 范围 | 命中文件数 | 命中次数 |
| --- | ---: | ---: |
| `app/main` | 37 | 557 |
| `app/renderer/src/main/src` | 1000 | 29401 |
| `app/renderer/src/main/public` | 42 | 197 |
| `app/renderer/engine-link-startup/src` | 108 | 2128 |
| `packageScript` | 7 | 20 |
| `package.json` | 0 | 0 |

计数包含代码标识符、样式类、兼容键、测试文本和法律归属，不代表同等数量的用户可见品牌残留。

## 二、必须保留的内部兼容标识

| 类别 | 代表位置 | 保留原因 |
| --- | --- | --- |
| 通信与窗口通道 | `app/main`、两个渲染项目 | IPC 与窗口完成信号属于稳定接口，本阶段禁止改名 |
| 路由、枚举、类型和组件名 | 主渲染源码 | `YakitRoute`、类型、组件和样式类属于源码兼容标识 |
| 数据库、缓存与远程文件名 | `app/main/filePath.js`、渲染常量 | `default-yakit.db`、缓存键与远程认证文件属于数据协议 |
| 引擎握手与版本请求 | 两个渲染项目 | `Hello Yakit!`、`fetch-yakit-version` 和更新类别属于引擎兼容协议 |
| Yaklang 名称 | 两个渲染项目 | 引擎与语言的正式名称，不属于桌面产品显示名 |
| 引擎压缩包与系统模式文件名 | `packageScript` | 属于既有载荷协议，禁止在品牌阶段修改 |
| 旧导出符号 | 品牌调用附近的渲染代码 | 导出名保留以避免业务调用变化，实际默认图形已指向睿眼资源 |

## 三、法律与开源归属

| 残留 | 位置 | 保留原因 |
| --- | --- | --- |
| 上游项目贡献者名称 | `product/renyan.json` 版权字段 | 保留原权利归属，不删除法律声明 |
| 仓库名 `yakit` | Fork、问题、许可证与通知链接 | 仓库真实路径，不能改写为不存在的仓库 |
| 上游文档域名与 Yaklang 名称 | 菜单与产品配置 | 明确指向上游开源文档和引擎语言来源 |
| 原许可证正文 | 根 `LICENSE.md` 与打包法律资源 | 法律材料保持原文 |

| RY-KI-017 | 第三级导航能力当前没有正式三级交付项 | 外壳组件支持子项层级，目标菜单当前实际数据为一级组与二级入口，不虚构额外功能层级 |
