# 睿眼产品化决策记录

## 阶段一：产品身份与构建适配

## RY-D-001 单一产品配置

产品身份以 `product/renyan.json` 为唯一可编辑来源。应用主侧、两个渲染项目、打包配置与品牌生成脚本通过适配层读取，组件不得复制产品常量。

## RY-D-002 应用标识

当前采用 `io.github.sharptornadoqsh.renyan`。仓库没有发布者证书、受控域名或商店登记材料，因此该值属于分支阶段标识；正式签名或商店发布前必须依据发布者材料复审，不自行推定证书身份。

## RY-D-003 可执行文件与产物名

Windows 与 macOS 基础名使用 `RenYan-Pentest`，Linux 使用 `renyan-pentest`，产物前缀使用 `RenYan-Pentest`。用户可见显示名统一为“睿眼自动化渗透系统”。

## RY-D-004 独立用户数据

默认用户目录为系统应用数据根目录下的 `RenYan-Pentest`，项目目录为其下的 `projects`。不探测、不复制、不迁移旧产品数据；只有用户显式设置的 `YAKIT_HOME` 才能引用其他位置。

## RY-D-005 内部兼容名称

引擎握手、更新查询、通信通道、路由、数据库文件、缓存键、插件接口和引擎压缩包协议继续使用既有兼容名称。产品发布名与内部兼容名分离，避免品牌工作改变协议。

## RY-D-006 安装脚本

新打包配置不引用 `build/yakit_build.nsh`，防止旧目录迁移逻辑进入睿眼安装程序。历史文件保持原状，许可证正文与第三方通知不删除。

## RY-D-007 品牌资源

品牌资源由原创的眼形、六边形节点、定位框和线路图形构成，不引用上游标志。矢量源由脚本生成，Pillow 生成多尺寸 PNG、ICO 与 ICNS；生成脚本执行格式与尺寸自检。

## RY-D-008 法律材料

根 `LICENSE.md` 保持原文，打包配置提供许可证与 `product/legal/THIRD_PARTY_NOTICES.md` 入口。产品版权字段保留上游项目贡献者权利说明。根 `package.json` 的许可证字段与正文不一致，阶段一不自行修改，列入已知事项。

## RY-D-009 关于信息

主侧系统菜单和托盘使用本地关于窗口，主渲染帮助菜单使用本地关于界面。两者均显示版本、引擎、构建、版本类别与法律入口；外部旧品牌关于页面不再作为用户入口。

## RY-D-010 版本类别

社区版是产品配置的默认类别。具体构建类别由 `product/build.js` 解析后注入；企业版免许可证开发构建显示“企业版（免许可证）”，普通企业版显示“企业版”。

## RY-D-011 最终验证目标

用户最终指定企业版免许可证开发版本用于界面查看，因此最终构建命令为 `yarn build-renders-test-enterprise-no-license`，不继续生成安装程序。此前产生的 Windows 社区版诊断安装程序早于最终源码状态，不作为本次最终交付产物。

## RY-D-012 构建提交标识

开发构建内的提交标识为构建时 `HEAD`：`452abcef315dc343e3855d2ac3fbca50acbf1b93`。本次功能提交在验证后创建，因此开发产物中的标识代表验证基线，不等同于最终提交哈希；正式发布应从干净提交重新构建。

## 阶段二：应用外壳与导航

### RY-D-013 菜单元数据作为用户入口基线

睿眼导航以 `RENYAN_MENU_MODEL` 为单一菜单定义，节点显式记录键、路由、显示名称、分组、顺序、能力、交付状态、功能标志和子项。用户可见名称从该模型派生，旧路由枚举值保持不变。

### RY-D-014 隐藏与删除分离

序号十一、十五、十七和二十四采用默认关闭的功能标志从导航过滤；序号二十五不建立菜单。过滤只改变入口可见性，不删除路由、页面或内部调用，因此已有页面间跳转仍可使用原路由。

### RY-D-015 旧菜单实现保留

`PublicMenu` 与 `HeardMenu` 的旧实现保留为兼容分支，睿眼外壳通过一个本地开关选择新导航。该方案限制变更范围，也允许在动态验证发现关键跳转异常时快速恢复原导航进行定位。

### RY-D-016 首页只呈现可证明的数据

项目、流量和风险概览只读取已有查询结果。缺失值映射为未知或空状态，查询失败映射为错误状态，真实零值映射为无数据状态；不得以示意数量填充首页。引擎状态使用外壳已有连接状态，团队服务不产生数量。

### RY-D-017 交付状态不等同于功能可用

团队工作区和成员角色保留用户可见名称，但交付状态为 `planned`，导航不可用，显示文案固定为“待团队阶段实现”。相同规则适用于功能标志被显式开启但仍未交付的四个首期隐藏项。

### RY-D-018 外壳主题与业务主题分界

全局主色沿用产品配置确定的蓝色体系。`envfile.tsx` 负责按产品类别和浅深模式提供主色，`useTheme` 负责向文档根节点写入主色与成功、警告、错误状态色，确保页面、浮层和关于对话框共享变量；阶段二不逐页改写核心功能页面的局部样式与业务实现。

### RY-D-019 复用既有动作通道

本地项目、关于与引擎更新不建立新的进程间通信或远程过程调用。它们通过既有设置事件进入原项目管理、关于界面和更新弹层；顶部导航与侧边栏之间的纯界面组选择使用窗口自定义事件，不进入业务协议层。

### RY-D-020 内部路由键不作为产品文案

路由键继续用于页面缓存、导航和测试定位，但页面标题、面包屑、菜单与浏览器标题只显示睿眼名称映射。数据属性可保留内部键供程序定位，不在可见标题区呈现。
