# 客户端与引擎兼容清单

## 一、适用范围

本文件记录客户端版本、引擎版本边界、预置工件及摘要证据。运行时数据源为
`product/engine-compatibility.json`，安装包同时携带该清单。最低版本没有权威材料时保持 `TBD`，运行时以
`check-secret-local-grpc` 能力检查作为兼容判定依据，不以推定版本代替真实证据。

## 三、版本来源

| 数据 | 来源 | 使用方式 |
| --- | --- | --- |
| 客户端版本 | 根 `package.json` | 选择对应兼容清单记录 |
| 推荐引擎版本 | `product/engine-compatibility.json` | 预置版本声明与本地兼容提示 |
| 最低引擎版本 | 权威测试结果或发布材料 | 当前缺失，保持 `TBD` |
| 最高已验证版本 | 本地实体引擎与能力检查 | 当前为 `1.4.8-beta3` |
| 在线最新版本 | 既有上游版本接口 | 仅在主界面可用后执行后台查询，不参与启动阻塞 |
| 在线摘要 | 与下载地址对应的 `.sha256.txt` | 用户显式下载时获取并校验 |

## 四、运行时兼容规则

1. 用户选择远程模式时不检查或启动本地引擎。
2. 本地引擎存在时，先执行本地能力检查，再读取本地版本与兼容清单；这些操作不需要网络。
3. 能力检查失败时标记为不兼容，并提供预置恢复、显式安装、远程连接和诊断入口。
4. 能力检查通过且版本低于已知最低版本时阻断；最低版本为 `TBD` 时不执行未经证实的数值阻断。
5. 能力检查通过且低于推荐版本时继续启动；主界面可用后在后台查询更新。
6. 高于最高已验证版本但能力检查通过时允许启动，并保留超出已验证范围的诊断信息。

## 五、工件信任规则

预置恢复必须同时验证压缩包摘要与压缩包内引擎摘要。构建钩子会在打包前读取运行时清单；摘要为 `TBD`、实体文件缺失或摘要不一致时立即拒绝打包。在线安装使用临时文件，取得真实摘要并校验后才替换版本缓存和活动引擎。手工安装要求同目录存在同名 `.sha256.txt` 文件。
