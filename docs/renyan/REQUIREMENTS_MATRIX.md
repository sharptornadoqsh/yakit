# 睿眼最终需求验收矩阵

## 一、判定规则

本矩阵以《功能要求》工作簿的三十一项编号需求为范围，并把序号二十六之后的互联网服务附注单列为
“26-A（附注）”。状态仅使用“已完成并已查看”“已完成但未完整验证”“部分完成”“外部服务提供并已验证”
“外部服务提供但未完整验证”“暂不纳入首期”“外部依赖”“待第三方材料”“未完成”。界面入口存在、菜单
可见或空页面可见，均不单独构成“已完成并已查看”。

本次为开发模式轻量验收，页面查看只证明当前客户端的可见状态。真实代理、扫描、爆破、加密导入导出、
多用户共享、服务端权限强制、长时可靠性及第三方材料没有被当前查看替代。

## 三、汇总

三十一项编号需求中，“已完成但未完整验证”十五项，“部分完成”七项，“外部服务提供但未完整验证”
两项，“暂不纳入首期”五项，“外部依赖”一项，“待第三方材料”一项；“已完成并已查看”
“外部服务提供并已验证”和“未完成”均为零项。26-A 是附注材料项，不纳入三十一项编号统计。

十六项非星标功能需求由序号七至十八、二十一至二十四组成。首期纳入十二项，其中九项为“已完成但未完整
验证”，三项为“部分完成”，首期完成但未完整验证比例为 \(9/12=75\%\)。按全部十六项统计，该比例为
\(9/16=56.25\%\)；另有四项“暂不纳入首期”。

十四项星标编号需求中，六项为“已完成但未完整验证”，一项为“外部依赖”，两项为“外部服务提供但未
完整验证”，四项为“部分完成”，一项为“待第三方材料”。26-A 附注另列为“待第三方材料”。
