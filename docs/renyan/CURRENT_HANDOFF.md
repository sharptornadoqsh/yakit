# 睿眼前端产品级重构移交

## 当前结论

本次工作以 `功能要求.xls` 的三十一项需求和视觉目录中的六张页面设计图为基线，完成睿眼主渲染端的产品结构重组。两张设计图内容完全相同，因此实际参考五组独立设计板。改动覆盖全局命令栏、十组主导航、当前模块导航、统一页面标题、公共组件、工作台、账号与角色、配置管理、项目管理，以及此前已经调整的代理、流量、重放、检测、爆破、编解码、插件、风险和扫描页面。

业务页面继续使用原路由、页签缓存、状态存储、处理函数、进程间通信、远程过程调用、引擎调用和外部服务请求。没有建立演示页面、模拟接口或固定业务记录。无真实数据时使用空状态；无法证明完整交付的能力保持隐藏或计划状态。

## 全局产品结构

应用采用固定命令栏、主导航、当前模块导航和任务工作区组成的统一壳层。页面实例仍由原 `MainOperatorContent` 管理，壳层重组没有改变页签键或页面缓存位置。

睿眼模式的桌面标题栏现缩减为三十二像素极简窗口框，只保留拖动区域和当前平台的最小化、最大化、关闭控制。旧全局状态、模式入口、功能域、帮助、性能与引擎信息不再显示在该区域；非睿眼分支及原窗口关闭确认逻辑保留。

| 顺序 | 主导航 | 真实入口或页面内区域 |
| ---: | --- | --- |
| 1 | 工作台 | 安全概览、最近任务、团队动态、风险趋势 |
| 2 | 交互代理 | 代理控制台、劫持会话、拦截规则、证书与代理设置 |
| 3 | 流量中心 | 历史流量、请求详情、响应详情、流量筛选 |
| 4 | 漏洞检测 | 通用检测、专项检测、检测任务、风险结果 |
| 5 | 爆破测试 | 爆破任务、字典管理、命中结果、执行日志 |
| 6 | 报文工具 | 报文重放、报文差异、编解码、操作历史 |
| 7 | 插件中心 | 插件仓库、已安装插件、本地插件、插件开发、插件日志、插件配置 |
| 8 | 团队协作 | 服务连接、用户管理、角色权限、组织管理、团队动态 |

顶部命令栏提供全局功能搜索、新建任务、快速重放、环境管理、消息入口和当前用户信息。这些操作分别调用原页面打开函数或原事件总线，不使用独立替代流程。消息未读标记没有真实来源时不会显示；工作区标签不声明未经查询的引擎连接状态。

## 公共组件

`components/renyanUI/` 现包含以下可复用构件：

- 外壳：`RuiYanAppShell`、`RuiYanTopCommandBar`、`RuiYanPrimaryNav`、`RuiYanSecondaryNav`。
- 页面：`RuiYanPage`、`RuiYanPageHeader`、`RuiYanToolbar`、`RuiYanSplitPane`、`RuiYanDetailPanel`。
- 操作：`RuiYanButton`、`RuiYanIconButton`、`RuiYanTabs`、`RuiYanStatusBadge`。
- 数据：`RuiYanCard`、`RuiYanPanel`、`RuiYanMetricCard`、`RuiYanDataTable`、`RuiYanFilterPanel`、`RuiYanFormSection`。
- 浮层与状态：`RuiYanDrawer`、`RuiYanModal`、`RuiYanEmptyState`、`RuiYanLoadingState`、`RuiYanErrorState`。
- 资源：`RuiYanIcon` 的二十四枚原创界面图标，以及 `assets/renyan/` 内的连接、流量、请求、扫描、插件、风险、错误和离线矢量图。

这些组件使用独立元素结构和样式模块。弹窗与抽屉具有对话框角色、标题关联、退出键关闭、焦点循环和焦点恢复；页签具有页签列表、选中状态、面板关联及方向键切换。

## 真实功能保持

- 工作台继续读取 `GetCurrentProjectEx`、`QueryHTTPFlows`、`QueryAvailableRiskLevel` 和 `QueryHybridScanTask`；七日趋势、协议分布、风险态势与最近任务均由真实返回值计算，团队动态无查询契约时显示空状态。
- 交互代理继续使用原代理启动、拦截、放行、丢弃、修改转发、证书和规则调用。
- 历史流量、重放、通用检测、专项检测和爆破继续使用原查询、任务流、暂停、恢复、停止与结果组件。
- 差异对比继续使用原编辑器实例与进程间通信，并增加真实文本模式、UTF-8 字节模式、长度与差异位置摘要；交换、清空、上一处、下一处和换行仍绑定编辑器行为。
- 编解码、插件管理和插件开发继续调用原编辑器实例、进程间通信和远程过程调用。
- 用户管理形成组织、用户列表、用户详情三栏结构，继续读取原账号与组织接口；角色权限页面继续读取原角色列表，并以 `roles/detail` 返回值展示真实权限详情。
- 项目管理继续执行原项目查询、新建、编辑、删除、切换、导入和导出调用；四个主要操作入口已改为键盘可操作按钮。
- 安全设置继续读取和保存原全局网络、证书、代理、第三方应用与运行参数。

## 功能边界

安全测试报告、高级国密独立入口、插件链路编排和受管客户端总览默认关闭。它们的底层路由或已有实现没有删除。团队动态显示计划状态且不可导航，不创建静态协作记录。

账号、组织和角色页面依赖仓库外服务。前端入口与调用存在不代表服务端权限、多人共享、密码保存和审计要求已经验证。功能表中的检测报告、算法规范材料和已知漏洞修复材料仍受现有证据文档边界约束。

## 验证边界

依照本次目标约束，不启动桌面应用、开发服务或浏览器，不执行构建、打包、全量测试、全量类型检查和完整规则检查。实施期间只对目标文件进行 TypeScript 语法转换、Sass 解析、导入使用与路径解析、菜单模型、纯函数、事件绑定、矢量资源和引用关系检查。最终版本库检查结果在交付汇报中列出。
