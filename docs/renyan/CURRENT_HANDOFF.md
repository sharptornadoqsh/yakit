# 睿眼自动化渗透系统阶段八移交

## 一、阶段结果

阶段八完成账号服务接入路径的限定安全处理。客户端保持 MD5 登录协议和授权请求头格式不变，恢复 HTTPS 证书验证，停止保存私有域密码，完善令牌失效清理，固定密码复杂度边界，并使用操作系统安全存储保护远程引擎当前凭据。

本阶段没有修改外部服务协议、创建账号服务、执行软件打包、开展全量安全扫描或进入下一阶段。

## 二、代码结果

- `app/main/httpServer.js` 的 HTTPS 直连与代理均验证证书，认证失效时立即清除主进程令牌。
- `app/main/securityCredentials.js` 与 `app/main/localCache.js` 仅为远程引擎当前凭据提供安全存储；不可安全使用时不保存密码。
- `ConfigPrivateDomain.tsx` 不再保存原始密码或 Base64 密码，读取旧配置时删除旧 `pwd`，登录结束后清除密码输入。
- 两套 `RemoteEngine.tsx` 不再把密码写入引擎历史，切换本地模式时清除当前远程凭据。
- `passwordPolicy.ts` 统一八至二十位、大小写字母、数字、特殊字符和完整字符串边界。
- 外部地址继续由主进程限制为 HTTP 与 HTTPS，私有域使用 HTTP 时显示明文传输警示。

## 三、协议与会话结论

- 登录继续使用 `POST /api/urm/login`，`pwd` 仍为客户端生成的 MD5 值。
- 客户端没有把原始密码、MD5 值或账号令牌写入普通文本配置或 `localStorage`。
- 令牌原值继续写入 `Authorization`，没有增加 `Bearer` 前缀。
- 用户退出、认证失效、服务地址变化和新账号登录会移除或替换旧会话。
- MD5 值仍具有可重放风险，面向互联网部署必须使用 HTTPS，协议升级需客户端与外部服务端同步完成。

## 五、本地验证

四个定向测试文件共二十五项用例通过，覆盖安全缓存正常与失败分支、旧明文凭据清理、会话令牌清理、外部地址协议限制和密码复杂度边界。主侧脚本语法、三个语言资源解析、有界敏感模式检索和差异空白检查通过。

用户随后指定执行 `yarn build-renders-test-enterprise-no-license`，双渲染定向构建以零退出码完成；输出只有既有样式顺序、浏览器数据和大分块警告，没有编译错误。以 `ELECTRON_IS_DEV=0` 启动当前工作树后，窗口直接显示睿眼启动配置页，没有许可证激活页；本地引擎启动后到达私有服务登录页，密码控件保持隐藏状态，HTTP 服务地址警示可见。

动态查看没有填写或提交账号密码，也没有调用外部服务。业务主页面、登录后密码清空、退出令牌清理、地址变化后的令牌隔离、低权限用户与角色操作、真实错误证书拒绝仍分别依赖外部账号、服务环境或受控证书场景，不能由本次只读界面查看证明。
