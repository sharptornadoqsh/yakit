# 睿眼自动化渗透系统前端品牌清理移交

## 一、交付结论

独立初始化引导界面已从正常启动路径移除。主窗口在渲染就绪后直接显示，原初始化渲染端继续在隐藏窗口中承担引擎存在性检查、版本检查、本地启动、远程连接、错误恢复、重试和日志入口等职责。用户从系统内主动切换引擎时，既有引擎管理窗口仍可显示，因此没有删除初始化逻辑或改变通信协议。

允许范围内的用户可见旧产品名称、旧标志资源、引擎显示名称以及下载和导出默认文件名已经完成分类处理。英文短品牌统一为 `RuiYan`，中文产品全称保持“睿眼自动化渗透系统”，技术语言、固定目录、兼容握手、通信标识与开源归属保持原值。

## 二、启动流程修改

- `app/main/index.js` 将初始化窗口设为默认隐藏并从任务栏移除，主窗口在 `ready-to-show` 时显示。
- 普通刷新与清缓存刷新继续显示主窗口，初始化窗口只在后台重新载入。
- `engineLinkWin-done`、`yakitMainWin-done`、引擎渲染就绪通知和数据传递路径保持原状。
- `StartupPage` 取消右侧装饰面板、旧标志引用和非必要宽度过渡；状态、错误、重试、日志与手动配置能力继续存在。

开发模式日志记录主窗口就绪时 `isVisible: true`，初始化窗口就绪时 `isVisible: false`；系统中只有标题为“睿眼自动化渗透系统”的可见桌面窗口。两个渲染端均完成页面加载，没有出现白屏或致命加载错误。

## 三、品牌与文件名称修改

用户可见内容覆盖启动状态、远程引擎、更新、插件、智能代理、远程过程调用服务、在线判定、漏洞结果、历史会话、交互式终端、错误提示与三种语言资源。旧产品名称按上下文改为“睿眼自动化渗透系统”“睿眼”或 `RuiYan`，引擎显示名称改为 `RuiYan Engine`。

用户主动生成文件统一采用 `RuiYan` 前缀，并保留原扩展名与时间戳规则。主要名称包括：

- `RuiYan-MITM-CA.pem` 与 `RuiYan-MITM-GM-CA.pem`
- `RuiYan-MITM-Replacer-Rules-Config.json`
- `RuiYan-HTTP-Flows-<时间戳>.har`
- `RuiYan-History-<时间戳>` 与 `RuiYan-WebFuzzer-<时间戳>`
- `RuiYan-Risks-<时间戳>`、`RuiYan-Screenshot-<时间戳>.png`
- `RuiYan-Codec-Output-<时间戳>.txt` 与 `RuiYan-Diagram.png`
- 通用数据导出入口生成的 `RuiYan` 前缀文件名

证书内容、编码、信任流程、导出数据结构和业务接口均未改变。
