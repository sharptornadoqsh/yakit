# 睿眼账号服务安全控制说明

## 一、范围与结论边界

本文记录阶段八对客户端账号服务接入路径的限定核查结果，覆盖需求序号二十六至三十一。结论只适用于当前 Electron 仓库和本次可复查代码，不代表外部账号服务、数据库、部署环境或第三方检测已经满足采购要求。

## 二、登录摘要协议

外部账号服务当前要求 `POST /api/urm/login` 的 `pwd` 字段使用客户端生成的 MD5 值。客户端继续在提交登录请求前生成该值，没有单方面改变算法、字段或服务协议。

客户端 MD5 不等于安全的密码存储。MD5 值可以被复制并作为可重放凭据使用，必须通过 HTTPS 防止传输窃听。阶段八停止把原始密码以 Base64 形式保存到远端配置，停止把密码写入在线配置，并在登录完成或失败后清除表单密码。登录失败提示不再展示服务返回的原始错误内容，避免密码或 MD5 值被服务错误信息带回界面。

后续密码协议升级必须由客户端和外部服务端同步设计、发布与验证。外部服务端的密码派生、加盐、数据库字段和历史迁移方式未在当前仓库中出现，仍需服务端源码、数据库结构或正式检测材料确认。

## 三、HTTPS 与证书校验

`app/main/httpServer.js` 的直连 HTTPS 代理和上游代理均设置 `rejectUnauthorized: true`。证书链错误不再被默认忽略，仓库内没有为生产路径提供关闭全部证书校验的选项。

内网自签名证书应在进程启动前通过 Node 标准环境变量 `NODE_EXTRA_CA_CERTS` 指向受控 CA 文件。该方式只增加明确证书，不允许忽略未知证书错误。私有域表单仍允许 HTTP 地址以维持现有部署兼容，但会显示明文传输警示；面向互联网部署必须使用 HTTPS。

## 四、令牌保存、发送与清理

- 登录令牌保存在主进程 `USER_INFO` 和当前渲染进程状态中，没有写入 `localStorage`、普通文本配置或本地缓存。
- 请求层按照外部服务协议将令牌原值写入 `Authorization`，没有增加或删除 `Bearer` 前缀。
- 退出登录调用 `resetUserInfo` 清除主进程令牌；渲染进程同步退出并清除当前用户状态。
- 服务返回认证失效时，主进程立即清除活动令牌，只向渲染进程返回不含令牌的退出所需用户信息。
- 私有域根地址变化时立即清除旧会话，避免一个服务地址的令牌发送到另一个地址。
- 新账号登录会替换当前进程状态；旧令牌不进入持久化存储。
- HTTP 错误日志使用 `pickAxiosErrorCore`，不记录请求头和请求体，因此不包含完整 `Authorization` 或登录密码。

令牌没有长期持久化需求，因此阶段八没有为账号令牌新增 `safeStorage` 存储。

## 五、密码存储状态

客户端私有域配置只保存服务根地址、代理、账号名和服务类别，不再保存原始密码或 MD5 值。读取旧配置时如发现 `pwd` 字段，会删除该字段并改写配置。在线配置写入明确使用空密码字段，防止旧值继续保留。

远程引擎密码不属于外部账号密码，但同属敏感凭据。当前远程连接使用主进程 `safeStorage` 加密后进入本地缓存；系统安全存储不可用、加密失败，或 Linux 使用 `basic_text` 后端时，只保存非敏感连接参数。引擎历史连接写入空密码，读取旧历史时也以空密码改写。切换本地引擎会清除当前远程凭据缓存。

外部服务端的用户密码存储方式未确认。需求序号三十的当前结论为：“客户端满足，服务端待确认。”

## 六、密码复杂度

客户端统一检查密码长度为八至二十位，并要求同时包含大写字母、小写字母、数字和特殊字符。完整字符串边界已经固定，不能再由合规前缀掩盖超长、空白或其他非法后缀。该规则用于服务登录表单和现有密码变更表单。

新建用户和重置密码由外部服务生成密码，当前客户端没有输入新密码的字段。当前没有外部服务测试账号用于直接调用 `/api/urm/reset/pwd` 或绕过前端提交弱密码。需求序号二十九的当前结论为：“客户端存在复杂度提示，外部服务端强制校验待确认。”

## 七、用户与角色权限

客户端存在 `/api/urm`、`/api/urm/edit`、`/api/urm/reset/pwd`、`/api/roles` 和 `/api/roles/detail` 调用，管理员页面具有前端角色门槛。前端隐藏菜单只控制界面显示，不能证明外部服务端实施授权。

阶段八没有低权限测试账号，没有执行用户编辑、密码重置、角色创建、编辑或删除。需求序号二十八的当前结论为：“客户端权限展示已实现，服务端接口权限待外部服务验证。”

## 八、日志与外部地址

有界检索没有发现阶段范围内记录原始密码、MD5 值、完整令牌、授权请求头、Cookie、私钥或远程引擎密码的日志调用。登录失败信息使用通用提示，不直接展示服务原始错误。

主进程 `normalizeHttpUrl` 只允许 `http:` 和 `https:`。`file:`、`javascript:`、`data:`、`shell:` 及未知自定义协议均被拒绝，外部页面打开处理复用该校验。

## 十、未验证事项

- 外部服务端密码存储、加盐、摘要和数据库结构。
- 弱密码通过前端、直接接口和初始密码设置路径时的服务端拒绝效果。
- 低权限账号对用户与角色接口的查看、编辑、重置和管理限制。
- 面向互联网部署的真实 HTTPS 证书、TLS 配置和第三方安全检测报告。
- 产品及全部第三方组件在指定日期的完整 CVE、CNVD 清单与修复公告。
- macOS、Linux 与不同安全存储后端的运行效果。
